Push to Repository

How to share your AI skills with the team using Claude

Combinate | April 2026

Overview

What does this skill do?

One simple prompt

Just say "Push this to the repository" and Claude handles everything automatically.

Creates a pull request

Your work is packaged into a branch and submitted to GitHub for review before it goes live.

Notifies the team

A notification is posted in #executive-assistant on Slack, tagging Jim for review.

Setup

What do you need to get started?

GitHub Access Key

A personal key that lets Claude push your work to GitHub on your behalf. You generate this yourself in a few steps.

Slack Bot Token

This is already provided for you. You just need to save it to your machine once using Claude. No setup required.

That's all you need

Claude automatically sets up everything else. You only need to provide your GitHub key once, and Claude saves it to your machine for future use.

Step 1

Get your GitHub Access Key

1
Go to github.com and log in
2
Click your profile photo
Top right corner
3
Settings → Developer settings → Personal access tokens → Tokens (classic)
4
Generate new token (classic)
Name it "Combinate Assistant", expiry 90 days
5
Tick the "repo" checkbox, then click Generate token
6
Copy the key immediately
You will not see it again

Your key will look like this

ghp_aBcDeFgHiJkLmNoPqRsTuVwXyZ123456

It always starts with ghp_ followed by a long string of letters and numbers.

Step 2

Save your keys using Claude

Open Claude Code in the Executive-Assistant folder and type these prompts exactly.

GitHub Key
Save this GitHub token to my .env file: ghp_your_key_here

Replace ghp_your_key_here with your actual key from Step 1.

Slack Token Provided for you — copy as-is
Save this Slack bot token to my .env file: xoxb-2551073081-10784047718724-fOuiIPFKlgw74BBk3fEDhI3R

This is the shared Combinate Claude bot token. Do not share it outside the team.

Step 3

Run your personal setup

Open Claude Code in the Executive-Assistant folder and type:

run setup

Saves your preferences

Claude asks a few simple questions about your role and how you work. Takes about one minute.

Private to your machine

Your personal setup is never shared with the team or committed to the repository.

One time only

Once done, you never need to repeat this. Claude remembers your setup for future sessions.

Troubleshooting

Something not working?

GitHub push fails

Your GitHub key may have expired. Keys last 90 days. Generate a new one using Step 1 and tell Claude to update it.

Slack notification not sending

Repeat Step 2 and re-save the Slack token. If the problem continues, ask Jim or Shane.

Nothing to commit

Claude will warn you if there are no saved changes. Make sure your skill files are saved before running the prompt.
