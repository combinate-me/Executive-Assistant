Version 1.3.0

Two issues we're addressing

Issue 1: Webhook Failures

Ecommerce webhooks are not firing correctly. Root cause: IMMS is on Ecommerce Module v5.8.0 which has a known webhook bug.

High impact — affects order processing

Issue 2: 504 Timeout Errors  TICKET 30904

Intermittent 504 Gateway Timeout errors for users and backend processes. Root cause: database queries not optimised for current data volume.

High impact — intermittent platform disruption
Option 1

Ecommerce Module Update v5.8.0 → v5.11.2

What we do

  • Update Ecommerce module to latest version
  • Update all affected front-end & back-end code
  • Migrate existing data (payment_method table)
  • Update APIs from V1 to V2 endpoints

Resolves

Webhooks — resolved 504 errors — partial (Ecommerce only)

Risks

  • Data migration on payment_method table (historically complex)
  • Risk of knock-on effects to connected databases
  • Non-Ecommerce 504s need separate work

Option 2 Our Recommendation

Targeted Fix + GraphQL Optimisation

What we do

  • Insites hotfix for Ecommerce v5.8.0 (webhooks fixed without full upgrade)
  • Optimise GraphQL queries platform-wide
  • Back-end: remove total_entries from APIs
  • Add retry logic for 504 errors on critical functions
  • PlatformOS database indexing for remaining timeouts

Resolves

Webhooks — resolved 504 errors — full platform

Consideration

  • Ecommerce stays on v5.8.0 for now
  • Module upgrade deferred, not cancelled — can be scoped as a future version

Our recommendation

We recommend Option 2

  • 1
    Speed to resolution — Issues fixed in ~4 days versus 3–4 weeks. 504 errors are actively impacting the platform now.
  • 2
    Lower risk — No data migration means no risk of disruption to payment data or connected databases.
  • 3
    Cost efficiency — Option 2 is approximately 25% of the cost of Option 1.
  • 4
    Broader 504 fix — Resolves timeouts across the entire platform, not just Ecommerce-related queries.

The Ecommerce module upgrade is not off the table. It can be scoped as a future version once the platform is stable.
